# v1.2.2-beta.2

Release date: 2026-01-17

## Changes

### Concurrent Background Task Execution

Implemented a proper concurrency control system for background agents:

**Features:**
- Semaphore-based concurrency limiting per provider
- Global concurrency limit (default: 10 concurrent tasks)
- Per-provider limits (DeepSeek: 5, ZhiPu: 5, MiniMax: 3)
- Automatic task queuing when limits are reached
- FIFO queue processing when slots free up

**StatusLine Enhancement:**
```
omc ● ready (0/10)                                      # Idle - 0 of 10 slots used
omc [⠙ Oracle: 32s DS/R] "Analyze the code..." (1/10)  # One active task
omc [⠙ Oracle: 32s] [⠹ Lib: 12s] (2/10)                # Multiple parallel tasks
omc [⠙ Oracle: 32s] (5/5 +2q)                          # At limit with 2 queued
```

**Configuration:**
```json
{
  "concurrency": {
    "global": 10,
    "per_provider": {
      "deepseek": 5,
      "zhipu": 5,
      "minimax": 3
    }
  }
}
```

### How Parallel Execution Works

1. Call `launch_background_task` multiple times (returns immediately)
2. Tasks run in parallel on the MCP server
3. Use `poll_task` to check results later
4. Concurrency limits prevent resource exhaustion

### Bug Fixes

**Fixed: Regular install now updates existing command files**

Previously, `bun run install-local` would skip updating command files if they already existed. Now oh-my-claude's command files are always updated on reinstall, ensuring you get the latest commands.

**Fixed: Force install preserves statusline wrapper**

Previously, `--force` reinstall would lose the CCometixLine (or other third-party statusline) wrapper, leaving only omc's statusline. Now the wrapper is preserved and regenerated with the updated omc statusline command.

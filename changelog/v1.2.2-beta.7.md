# v1.2.2-beta.7 - 2026-01-23

## Session Segment - Quota Usage Only

The session segment now shows **only API quota usage** (no cost data).

### OAuth Credential Sources

Following CCometixLine's approach, credentials are read from:

1. **macOS**: Keychain via `security find-generic-password -a <user> -w -s "Claude Code-credentials"`
2. **Other OS**: File at `~/.claude/.credentials.json`

### Display Format

- **Primary**: 5-hour utilization (e.g., `79%`)
- **Secondary**: 7-day utilization (e.g., `7d:4%`)

### Color Thresholds

- 🟢 Green: <50% usage
- 🟡 Yellow: 50-80% usage
- 🔴 Red: >80% usage

### Example Output

```
omc [opus-4.5] [dev*↑2] [oh-my-claude] [45% 89k/200k] [79% 7d:4%] [eng-pro]
                                                       └── 79% 5-hour, 4% 7-day quota
```

### Fallbacks

1. **Custom API**: Shows `0%` for non-Anthropic API endpoints
2. **No credentials**: Shows `[?]` gracefully

### Technical Changes

- `src/statusline/segments/session.ts`:
  - Added `getOAuthToken()` to read from Keychain (macOS) or credentials file
  - Added `getOAuthTokenMacOS()` using `security` command
  - Added `getOAuthTokenFromFile()` for `~/.claude/.credentials.json`
  - **Removed cost display** - quota usage is now the only data shown
  - Removed unused `formatCost()` and `formatDuration()` functions

- `src/statusline/statusline.ts`:
  - Extended `parseClaudeCodeInput()` to include `cost`, `workspace`, `oauth` fields
  - Increased timeout from 100ms to 5s (allows keychain + API calls, cached results fast)

### Caching

API responses cached at `~/.claude/oh-my-claude/cache/api_usage.json` for 5 minutes to avoid repeated API calls.

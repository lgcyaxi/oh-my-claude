# v1.2.2-beta.3

Release date: 2026-01-17

## Changes

### Non-blocking MCP Commands for Animated Statusline

Updated `/omc-oracle` and `/omc-librarian` commands to use non-blocking task execution pattern:

**Before (blocking):**
```
execute_agent → blocks until complete → statusline frozen at ⠙ 0s
```

**After (non-blocking with wait):**
```
launch_background_task → poll_task(wait_seconds: 30) → statusline animates
```

This allows the statusline spinner and timer to update in real-time while waiting for agent responses.

### New: poll_task wait_seconds Parameter

Added `wait_seconds` parameter to `poll_task` to reduce polling noise:

```
poll_task(task_id: "...", wait_seconds: 30)
```

- Waits up to N seconds before returning (capped at 60s)
- Reduces the number of visible tool calls while maintaining statusline animation
- Internal polling every 500ms during wait period

### Bug Fixes

**Fixed: Invalid agent reference in AGENT_ABBREV**

The statusline formatter had a reference to non-existent "explore" agent. Fixed mapping:
- Removed: `explore: "Exp"` (agent doesn't exist)
- Added: `analyst: "Analyst"` (correct agent)
- Added: `prometheus: "Prom"` (missing agent)

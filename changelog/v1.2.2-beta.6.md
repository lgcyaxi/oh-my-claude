# v1.2.2-beta.6 - 2026-01-23

## Session & Context Segment Improvements

Clarified and improved the statusline segments to properly distinguish between quota usage and token usage.

### Session Segment - Quota/Usage Display

The session segment now shows **API quota usage** (like CCometixLine's Usage segment):

**Data Priority:**
1. **Cost Data** (Primary): Shows session cost from Claude Code stdin
   - Display: `$1.23 45m +100 -50` (cost, duration, lines changed)
   - Colors: green (<$1), yellow ($1-5), red (>$5)

2. **OAuth API Usage** (Fallback): Fetches 5-hour/7-day utilization
   - Display: `75% 5h:30%` (7-day usage, 5-hour usage)
   - Cached for 5 minutes at `~/.claude/oh-my-claude/cache/api_usage.json`
   - Colors: green (<50%), yellow (50-80%), red (>80%)

3. **Graceful Degradation**: Shows `[?]` when no data available

### Context Segment - Token Usage Display

The context segment now shows **token usage** (context window percentage):

**Features:**
- Parses Claude Code transcript JSONL from `transcript_path`
- Supports multiple provider formats (Anthropic, OpenAI)
- Normalizes token usage including cache tokens
- Display: `45% 90k/200k` (percentage, current/limit)

**Token Normalization:**
```
total = input + output + cache_read + cache_create
```

**Context Window Limits:**
- Claude 4.5/3.x: 200k tokens
- GLM-4: 128k tokens
- Qwen: 256k tokens

**Color Thresholds:**
- green: <50% usage
- yellow: 50-80% usage
- red: >80% usage

### Types Extended

Added new fields to `ClaudeCodeInput` interface:
- `cost` - Session cost and duration data
- `workspace` - Current working directory
- `oauth` - API base URL and access token for usage fetch

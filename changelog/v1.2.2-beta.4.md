# v1.2.2-beta.4 - 2026-01-18

## Fixed

### Windows Compatibility
- **Critical**: Fixed Windows "Open with" popup dialogs for all Node.js scripts
  - MCP server now uses full path to `node.exe` instead of just `node`
  - Hook scripts now use full path to `node.exe`
  - Statusline wrapper now uses full path to `node.exe`
  - Statusline wrapper changed from `.js` to `.mjs` (ES modules) for proper execution

### Root Cause
Windows was trying to execute `.js`/`.mjs` files directly without Node.js, causing file association dialogs. This affected:
- `oh-my-claude-background` MCP server
- Hook scripts (`comment-checker.js`, `todo-continuation.js`, `task-tracker.js`)
- Statusline wrapper script (`statusline-wrapper.mjs`)

### Changes
- `src/installer/settings-merger.ts`: Added `getNodeCommand()` helper to resolve full node path
- `src/installer/statusline-merger.ts`: Added `getWrapperCommand()` helper, changed to `.mjs`
- All installer commands now use absolute paths on Windows (e.g., `"D:\IDKs\nodejs\node.exe"`)

## Technical Details

### Before
```json
{
  "command": "node",
  "args": ["C:\\Users\\.claude\\oh-my-claude\\mcp\\server.js"]
}
```

### After
```json
{
  "command": "\"D:\\IDKs\\nodejs\\node.exe\"",
  "args": ["C:\\Users\\.claude\\oh-my-claude\\mcp\\server.js"]
}
```

This ensures Windows can find and execute Node.js without relying on file associations.

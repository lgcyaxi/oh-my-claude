# v1.2.0

Release date: 2026-01-17

## Breaking Changes

### Removed Fallback System
- Removed automatic fallback to Claude models when external provider API keys are not configured
- MCP background agents now require their respective API keys to function
- Simplifies the architecture by removing the FallbackRequiredError handling

## Features

### New `analyst` Agent
- Added new MCP background agent using DeepSeek Chat for quick code analysis
- Faster than Oracle, suitable for simpler reasoning tasks
- Configuration: `deepseek/deepseek-chat` with temperature 0.1

### New `execute_agent` MCP Tool
- Added blocking variant of `launch_background_task` for cleaner UX
- Waits for agent completion and returns result directly
- Supports timeout parameter (default 5 minutes)
- Useful when you need the result immediately rather than polling

## Improvements

### StatusLine Enhancements
- Fixed per-session tracking to prevent cross-session interference
- Added provider display for MCP agents
- Walk process tree to find Claude Code PID for accurate session identification

### CLI Improvements
- Added `cleanup` command to remove stale session data
- Added PPID-based session discovery for better session tracking

## Migration Notes

If you were relying on the fallback system, you now need to:
1. Configure API keys for providers you want to use, or
2. Use Task tool agents (Sisyphus, claude-reviewer, claude-scout) which use Claude subscription

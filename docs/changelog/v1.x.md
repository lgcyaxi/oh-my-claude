# v1.x Changelog

The v1.x series built oh-my-claude from an initial multi-provider MCP server into a full-featured development platform with live model switching, semantic memory, and rich CLI tooling.

## v1.5.x (2026-02-05 — 2026-02-13)

- Three-tier semantic memory search: Hybrid (FTS5 + vector), FTS5-only, and legacy in-memory fallback
- SQLite index engine using sql.js-fts5 WASM with embedding providers (custom/zhipu/openrouter)
- Deduplication system (exact hash skip + semantic near-dupe detection) and snippet-only recall with `get_memory` drill-down
- Memory timeline (`TIMELINE.md`), memory clear/summary commands, and doctor memory health checks
- `cc` command with `--provider` direct mode, Kimi provider, per-session proxy isolation, route directives

## v1.4.x (2026-02-05)

- Live Model Switching Proxy (ports 18910/18911) with SSE streaming, signal file IPC, and auto-revert
- `/omc-switch` command, proxy MCP tools, proxy CLI, and proxy statusline segment
- OAuth proxy support and project-scoped memory with compaction
- Context-aware auto-save hook, memory awareness hook, and session prompt logging
- npm `dist/` packaging fix and Windows compatibility improvements (where, root dir, hook paths)

## v1.3.x (2026-01-29)

- `execute_with_model` MCP tool for model-specific task execution
- Markdown-first memory system with YAML frontmatter and 5 MCP tools (remember, recall, forget, list, status)
- Output Style Manager with 5 built-in presets (concise, detailed, technical, casual, formal)
- Memory statusline segment and memory CLI commands

## v1.2.x (2026-01-17 — 2026-01-29)

- Segment-based statusline architecture with configurable segments and presets
- Rich context system and concurrent background tasks with semaphore
- YAML frontmatter for agent definitions and analyst agent
- Windows compatibility fixes (Node popup suppression, git timeout, stderr handling) and fallback system removal

## v1.1.x (2026-01-15 — 2026-01-16)

- StatusLine with real-time updates, ANSI colors, and animated spinners
- `setup-tools`, `config`, and `cleanup` CLI commands
- `execute_agent` MCP tool and beta channel support
- Bug reporting via `/omcx-issue` and Windows installer/shell fixes

## v1.0.x (2025-01-15)

- Initial release: multi-provider MCP server supporting DeepSeek, ZhiPu, and MiniMax
- 9 specialized agents with slash commands
- CLI tooling (install, uninstall, status, doctor) and hook system
- v1.0.1: self-update command, Windows `pathToFileURL` fix, auto-fallback to Claude

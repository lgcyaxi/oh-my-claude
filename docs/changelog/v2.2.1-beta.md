# v2.2.1-beta.2 - 2026-03-13

> OpenRouter switched to native Anthropic-compatible API

---

## Highlights

- **Native Anthropic Messages API** — OpenRouter now uses the Anthropic-compatible `/api/messages` endpoint instead of OpenAI-compatible format
  - Full Anthropic protocol support: thinking blocks, SSE streaming, proper stop_reason
  - Bearer token auth (`Authorization: Bearer`) enables routing to all providers including "stealth" (free models)
  - Previous `x-api-key` header forced routing to only "anthropic" provider, excluding free models

---

## Technical Changes

- **Provider Type** — Changed OpenRouter from `openai-compatible` to `anthropic-compatible`
- **Base URL** — Updated from `https://openrouter.ai/api/v1` to `https://openrouter.ai/api`
- **Auth Header** — Uses `Authorization: Bearer` instead of `x-api-key` for free model routing
- **No Format Conversion** — Anthropic-native format means no OpenAI→Anthropic conversion needed

---

# v2.2.1-beta.1 - 2026-03-12

> OpenRouter provider support with free models

---

## Highlights

- **OpenRouter Provider** — Added OpenRouter as a new provider with two free models:
  - `openrouter/hunter-alpha` — Hunter Alpha (free, 1T params, 1M context, agentic)
  - `nvidia/nemotron-3-super-120b-a12b:free` — Nemotron 3 Super (free, 120B hybrid MoE, 1M context)

---

## Features

- **OpenRouter Integration** — OpenAI-compatible API with base_url `https://openrouter.ai/api/v1`
- **Provider Aliases** — `or` and `openrouter` as shortcuts for provider switching
- **Environment Variable** — `OPENROUTER_API_KEY` for authentication

---

## Migration Notes

### From v2.2.0

No breaking changes. Simply add `OPENROUTER_API_KEY` to use OpenRouter:

```bash
export OPENROUTER_API_KEY=your-openrouter-api-key
```

## Bug Fixes

- **OpenRouter Duplicate Prefix in Menubar Tray** — Fixed session header showing `openrouter/openrouter/hunter-alpha` instead of `openrouter/hunter-alpha` by adding a `starts_with` guard to avoid double-prefixing when the model ID already includes the provider prefix
- **WezTerm New Tab Crash (find_git_bash Fix)** — Fixed broken `find_git_bash()` Lua function on Windows:
  - Fixed `h:read('*l')` only reading first `where git` line (which was mingw64 path that didn't match `cmd/git.exe` regex)
  - Added support for non-standard Git paths like `D:\Applications\Git\`
  - Added 3 derivation strategies (cmd/, mingw64/, usr/)
  - Fixed fallback from `pwsh.exe` to `powershell.exe` (more universally available)

# v2.2.1-beta.7 - 2026-03-16

> Session Management + Standalone Dashboard

---

## Features

- **Session Browser** — Browse and review Claude Code conversation history from the web dashboard:
  - **Project sidebar** — Lists all projects with session counts, sorted by last modified
  - **Session list** — Shows sessions with summary, first prompt, date, git branch badges, and message count
  - **Search/filter** — Client-side filtering by text across summaries, prompts, and branches
  - **Conversation viewer** — Chat-style renderer with collapsible thinking blocks, tool use blocks, and scroll-to-top
  - Two-layer data strategy: scans JSONL files directly (always works) with sessions-index.json enrichment (when available)
  - Fast metadata extraction reads only first ~30 lines per session file

- **Standalone Dashboard Server** — Lightweight control-only server on port **18920** (no proxy port, separate from proxy defaults):
  - `oh-my-claude proxy dashboard` — starts at `http://localhost:18920/web/`
  - `oh-my-claude proxy dashboard --stop` — stops the dashboard
  - `oh-my-claude proxy web` — alias for dashboard
  - Auto-starts when first `omc cc` session launches (if not already running)
  - Only 215KB bundle (vs 290KB for full proxy) — no auth, no switch state, no proxy routing

- **Per-session model switching from dashboard** — Switch page lists all active proxy sessions, lets you switch/revert each one independently via forwarding API (`/api/instances/:controlPort/switch|revert|status`)

- **Dashboard UI improvements** — Replaced "Mode" card with "Active Proxies" count and "Session History" link; active sessions show "Switch →" shortcut

<details>
<summary>Technical Implementation Details</summary>

- Backend: `src/proxy/control/sessions.ts` — 4 REST endpoints (`/api/sessions`, `/:folder`, `/:folder/:id`, `/:folder/:id/meta`)
- Dashboard: `src/proxy/dashboard.ts` — single `Bun.serve()` on port 18920, control API only
- Instance forwarding: `/api/instances/:controlPort/(switch|revert|status)` proxies requests to per-session proxies
- PID management: `~/.claude/oh-my-claude/dashboard.pid` (separate from proxy PID)
- JSONL parsing: `createReadStream` + `readline` for memory-efficient streaming of large sessions
- Frontend: `SessionsPage.tsx`, `ConversationPage.tsx`, rewritten `SwitchPage.tsx` with per-session control

</details>

# v2.2.1-beta.6 - 2026-03-15

> Proxy Instance Registry + Dashboard Session Aggregation

---

## Features

- **Proxy Instance Registry** — Per-session proxy instances now register themselves in a shared file (`~/.claude/oh-my-claude/proxy-instances.json`) on startup:
  - Heartbeat every 60s to keep entries fresh; stale entries (>5min) auto-cleaned
  - New `/api/instances` endpoint aggregates data from all running per-session proxies
  - Dashboard "Active Sessions" section shows real sessions from all `omc cc` instances with uptime, request count, and provider/model info
  - Dashboard proxy (default port 18911) skips self-registration — it's the viewer, not a participant
  - Instances deregister on graceful shutdown

<details>
<summary>Technical Implementation Details</summary>

- Shared registry file: `~/.claude/oh-my-claude/proxy-instances.json`
- Each instance registers with: sessionId, pid, startTime, provider, model, port, heartbeatTime
- Stale cleanup: entries with heartbeat >5min old are removed on each registration
- Dashboard (port 18911) excludes itself from the instance list
- Graceful deregistration on SIGTERM/SIGINT

</details>

# v2.2.1-beta.4 - 2026-03-13

> OpenRouter sanitization, conversation breakage mitigation, and 1M context window detection

---

## Features

- **OpenRouter Sanitizer Module** — Extracted `sanitizeOpenRouter()` to dedicated `src/proxy/sanitizers/openrouter.ts` file (matching `deepseek.ts` pattern)
  - **Thinking Block Stripping** — Removes `<thinking>` blocks from requests to prevent parsing errors on non-Anthropic models
  - **Tool Definition Capping** — Limits tool definitions to 64 entries to prevent "request too large" errors (Claude Code with MCP servers sends 90+ tools which overwhelms free/small models)
  - **Unsupported Content Type Stripping** — Filters `tool_reference` and other extended content types not supported by all models

## Bug Fixes

- **Duplicate Provider Prefix in Proxy Logs** — Fixed log lines showing `openrouter/openrouter/hunter-alpha` instead of `openrouter/hunter-alpha`
  - Created `displayModel()` helper in `src/proxy/handlers/display.ts`
  - Applied to all log lines in `switched.ts`, `directive.ts`, and `index.ts` handlers

- **Statusline Context Window Detection for 1M Models** — Fixed statusline showing `200k` instead of `1M` for Opus 4.6 with extended context:
  - Added `parseContextSuffix()` to detect `[1m]`, `[200k]` suffixes in model IDs
  - Updated `getContextLimit()` to prioritize explicit suffix over pattern matching
  - Claude Code appends these suffixes to signal actual context window size

## Conversation Stability

- **OpenRouter Conversation Breakage Mitigation** — Multiple fixes to prevent corrupted conversation state when switching to OpenRouter:
  - Thinking block stripping prevents non-Anthropic models from receiving content they cannot parse
  - Tool capping prevents request-too-large errors that break conversation continuity
  - These fixes address issues where switching to OpenRouter's hunter-alpha would break conversations even after reverting back to Claude

# v2.2.1-beta.2 - 2026-03-13

> OpenRouter switched to native Anthropic-compatible API

---

## Highlights

- **Native Anthropic Messages API** — OpenRouter now uses the Anthropic-compatible `/api/messages` endpoint instead of OpenAI-compatible format
  - Full Anthropic protocol support: thinking blocks, SSE streaming, proper stop_reason
  - Bearer token auth (`Authorization: Bearer`) enables routing to all providers including "stealth" (free models)
  - Previous `x-api-key` header forced routing to only "anthropic" provider, excluding free models

---

## Technical Changes

- **Provider Type** — Changed OpenRouter from `openai-compatible` to `anthropic-compatible`
- **Base URL** — Updated from `https://openrouter.ai/api/v1` to `https://openrouter.ai/api`
- **Auth Header** — Uses `Authorization: Bearer` instead of `x-api-key` for free model routing
- **No Format Conversion** — Anthropic-native format means no OpenAI→Anthropic conversion needed

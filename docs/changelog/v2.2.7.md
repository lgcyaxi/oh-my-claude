# v2.2.7 Changelog

## v2.2.7 - 2026-04-08

### Improvements

- **Qwen 3.6 Plus Default** — Updated default Aliyun model from qwen3.5-plus to qwen3.6-plus across all aliases, agent fallbacks, and categories. `omc -p aliyun` and the `q` switch shortcut now use qwen3.6-plus. qwen3.5-plus remains selectable in the model registry.

- **Claude 4.6 Agent Models** — Updated native agent model references from Claude 4.5 to 4.6: Sisyphus (opus), Claude-Reviewer (sonnet), Claude-Scout (haiku), Prometheus (sonnet). Registry entries and categories updated accordingly.

- **Dashboard Adaptive Polling** — Dashboard now backs off polling from 5s to 15s after 3 consecutive connection failures, reducing noise when the proxy is down. Resets to 5s on successful reconnection.

- **Bounded Response Cache** — Proxy control API response cache now has a 50-entry size limit with LRU eviction, preventing unbounded memory growth from unique cache keys.

### Fixes

- **Aliyun MiniMax M2.7 Removed** — Removed MiniMax-M2.7 from Aliyun model registry (not supported on Aliyun Coding Plan). MiniMax-M2.5 remains available via Aliyun hub.

- **Missing Statusline Mappings** — Added display entries for Claude Haiku 4.6 and Qwen 3.6+ across context window, formatter abbreviation, and model display segments.

- **CC Session Exits Immediately in IDE Terminals** — Fixed `omc cc` spawning Claude Code inside tmux that immediately exits `[exited]` when launched from Cursor/VSCode terminals. Root cause: inherited Claude env vars (`CLAUDE_CODE_ENTRYPOINT`, `CLAUDE_CODE_EXECPATH`, `CODEX_COMPANION_SESSION_ID`) caused the child Claude to detect a parent instance and refuse to start. All spawn paths (tmux inline, tmux detached, spawnSync, Windows batch) now strip these variables.

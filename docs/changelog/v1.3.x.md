# v1.3.x Changelog

## v1.3.0-beta.3 - 2026-01-29

### Added

- **`execute_with_model` MCP Tool**: Direct model routing without agent/category lookup
  - Specify explicit `provider` and `model` for token-efficient calls
  - Blocking execution with configurable timeout
  - Bypasses agent system prompts for lightweight use cases
- **`routeByModel()` Provider Function**: New routing path in `src/providers/router.ts`
  - Validates provider exists and is not `claude-subscription`
  - Reuses cached provider clients
- **Memory StatusLine Segment**: New `memory` segment showing memory count
  - Displays `[mem:N]` with green (has memories) or neutral (empty) color
  - Added to `full` preset, disabled by default
  - Toggleable via `oh-my-claude statusline toggle memory on`

### Changed

- Task manager now supports `providerName` + `modelName` as a third routing option
- StatusLine config schema updated with `memory` segment (position 8)
- CLI `statusline toggle` command accepts `memory` segment

## v1.3.0-beta.0 - 2026-01-29

### Added

- **Output Style Manager**: CLI commands to manage Claude Code output styles
  - `oh-my-claude style list` - List available built-in and custom styles
  - `oh-my-claude style set <name>` - Switch active output style
  - `oh-my-claude style show [name]` - Display style content
  - `oh-my-claude style reset` - Reset to Claude Code default
  - `oh-my-claude style create <name>` - Create custom style from template
- **Built-in Style Presets** (5 styles):
  - `engineer-professional` - SOLID/KISS/DRY/YAGNI principles
  - `agent` - Autonomous agent mode, minimal narration
  - `concise-coder` - Code-first, no explanations
  - `teaching` - Educational with concepts and trade-offs
  - `review` - Code review with severity levels
- **Style Deployment**: Installer now deploys built-in styles to `~/.claude/output-styles/`
- **StatusLine**: New abbreviations for built-in styles (agent, coder, teach, review)

## v1.3.0-beta.2 - 2026-01-29

### Added

- **Memory System**: Markdown-first persistent memory across Claude Code sessions
  - Storage in `~/.claude/oh-my-claude/memory/` (sessions/ and notes/ subdirectories)
  - Human-readable markdown files with YAML frontmatter (git-friendly, editable)
  - **MCP Tools** (5 tools):
    - `remember` - Store a memory with optional title, type, and tags
    - `recall` - Search memories by text query with relevance scoring
    - `forget` - Delete a specific memory by ID
    - `list_memories` - Browse memories with type and date filters
    - `memory_status` - Show memory store statistics
  - **CLI Commands**:
    - `oh-my-claude memory status` - Show memory stats
    - `oh-my-claude memory search <query>` - Search memories
    - `oh-my-claude memory list` - List stored memories
    - `oh-my-claude memory show <id>` - Show memory content
    - `oh-my-claude memory delete <id>` - Delete a memory
  - In-memory text search with weighted scoring (title 3x, tags 2x, content 1x)
  - No external dependencies (no SQLite, no FTS5 for MVP)

### Changed

- Installer creates memory directories during installation
- MCP server now registers 5 additional memory tools

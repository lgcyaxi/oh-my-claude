# v2.2.5 Changelog

## v2.2.5 - 2026-04-01

### Fixes

- **Dashboard Revert to Claude Not Working** — Fixed `forwardToInstance()` in the control API dropping the `?session=` query parameter when forwarding switch/revert requests to per-session proxy instances. The menubar correctly passed session IDs, but the dashboard did not, causing revert to reset global file-based state instead of the active session's in-memory state.
- **Dashboard Session View Refactored to Session-Centric** — Replaced port/instance-centric session selector with a flattened per-session view. Each Claude Code session is now displayed individually with its session ID, making it clear which session is being targeted for switch/revert operations. Previously showed proxy instances by control port, which was confusing when multiple sessions shared a proxy.

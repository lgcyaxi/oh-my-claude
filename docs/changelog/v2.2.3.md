# v2.2.3 Changelog

## v2.2.3 - 2026-03-31

### Breaking Changes

- **Removed Custom Codex ACP** — Replaced the entire custom Codex integration (~500+ files: app-server daemon, JSON-RPC protocol, approval policies, OAuth, viewers, statusline segment, CLI commands) with the official OpenAI Codex plugin (`openai/codex-plugin-cc`). The plugin is auto-installed during `omc install`. Manual install: `claude plugins add openai/codex-plugin-cc`.

### Features

- **Dashboard Provider Usage** — New `/api/usage` endpoint and "Provider Usage" section on Dashboard page. Shows real-time balance/quota for all configured providers (DeepSeek, ZhiPu, MiniMax, Kimi, Aliyun) with color-coded status cards.
- **MiniMax M2.7 Model Support** — Added MiniMax-M2.7 across all providers (MiniMax, MiniMax CN, Aliyun). Agent defaults updated to use M2.7 for documentation and writing tasks.
- **GLM-5.1 Model** — Added `glm-5.1` to ZhiPu and Z.AI providers in models registry.

### Fixes

- **EMFILE Fix: Dashboard Static File Serving** — Replaced `Bun.file()` with `existsSync` + `readFileSync` for all static asset serving. Prevents FD exhaustion on Windows (512 FD limit) under dashboard polling load.
- **EMFILE Fix: API Response Caching** — Added 5-second TTL response cache for heavy GET endpoints (`/api/memory`, `/api/sessions`, `/api/preferences`, `/api/instances`). Cache invalidates automatically on mutating operations.
- **Dashboard Compact Type Filter** — Compact operation now filters by memory type (default: notes only), matching MCP `compact_memories` behavior. Execute phase preserves source type.
- **Dashboard Summarize Type Filter** — Summarize operation now supports `type` filter, allowing session-only or note-only summarization.
- **Doctor Embedding Connectivity** — Two-step connectivity check: verifies server reachability first, then tests embedding endpoint with specific error messages.
- **Menubar Search Path Order** — Fixed `omc menubar` failing with "missing tauri.conf.json". Installed version now checked first.
- **Statusline Usage: Load `.zshrc.api`** — Statusline now loads API keys from `~/.zshrc.api` at startup. Fixes provider usage segments being hidden when tmux session doesn't inherit shell env vars.
- **Statusline Usage: `!auth` Indicator** — Kimi and Aliyun usage segments now show `!auth` (red) when credentials are expired instead of silently hiding.
- **Aliyun Auth Detection** — Handle `BailianGateway.Login.NotLogined` response (200 with success=false) and 302 redirect as auth-expired.
- **Kimi/Aliyun API-key Fallback** — Show "ok" indicator when API key is set but no OAuth credential.
- **CLI Version from package.json** — Version now loaded dynamically from `package.json` instead of hardcoded string, preventing version drift.
- **Installer Codex Version Selection** — Fixed semver sort for Codex companion script version discovery.

### Improvements

- **Sisyphus Codex Plugin Integration** — Updated all agent routing tables, delegation priorities, and coworker sections to use official Codex plugin commands (`/codex:rescue`) instead of removed `coworker_task(target="codex")`.
- **Mandatory Codex Audit** — Sisyphus now enforces `/codex:rescue` as a BLOCKING step before reporting completion when code was changed and Codex is available.
- **Review Gate Default** — `omc install` now enables the Codex review gate by default.
- **Dashboard Memory Type Selectors** — Added compact/summarize type dropdown selectors in Settings page.
- **Updated Agent Prompts** — All orchestration commands and agent definitions updated with Codex plugin references.

### Removed

- `src/coworker/codex/` — Full app-server daemon, protocol types, runtime, conversation management
- `src/coworker/codex-runtime.ts` — CodexCoworkerRuntime class
- `src/shared/auth/codex.ts` — OpenAI OAuth flow (`omc auth openai` removed)
- `/omc-codex` slash command, `omc m codex log` CLI command, codex statusline segment
- `codex` target from `coworker_task` MCP tool
- Codex review helpers (`review-helpers.ts`, `review-scoped-diff.ts`)
- OpenAI/Codex usage segment
- OpenRouter Hunter Alpha model (discontinued, replaced by `nvidia/nemotron-3-super-120b-a12b:free`)

# v2.2.4 Changelog

## v2.2.4 - 2026-04-01

### Fixes

- **Dashboard OMC Mem Duplicate Entries** — Fixed `findOmcProjectMemDirs()` reading the same project memory directory multiple times when multiple Claude project folders resolve to the same cwd. Added three-layer deduplication: backend cwd dedup (root cause), API response ID dedup (defense in depth), and frontend ID dedup (safety net). Previously caused all projects to show the same accumulated list and unreliable click behavior due to duplicate React keys corrupting DOM reconciliation.
- **Stale Update Banner After `omc update --beta`** — Fixed update notification persisting after successful beta update. `printUpdateBannerIfCached()` now re-validates cached `latestRef` against current beta ref before showing banner. Both `omc update --beta` and `install-local` now clear the update check cache after writing the new beta marker.
- **Dashboard Memory Operations Timeout** — Increased client timeout from 60s to 180s to match backend AI call timeout, preventing "signal is aborted without reason" errors during daily consolidation.
- **Dashboard Memory Operations Global Scope** — Added "Global only" option to memory operations scope dropdown. Previously only project-scoped memories were selectable; global memories (`~/.claude/oh-my-claude/memory/`) can now be targeted independently.

### Features

- **Dashboard Session Bulk AI Rename** — Added "Rename N unnamed" button on Sessions page for batch AI-renaming of sessions without summaries directly from the session list.
- **Dashboard Old Session Cleanup** — Added "Clean N old" button on Sessions page to delete session JSONL files older than 15 days. Cleans up both files and sessions-index.json entries.

# v2.2.2 - 2026-03-18

> Dashboard project discovery, global memory resolution, AI operation timeout, menubar fix

---

## Features

- **MiniMax M2.7 Model Support** — Added MiniMax-M2.7 to model registry across all providers (MiniMax, MiniMax CN, Aliyun). Updated agent defaults (`document-writer`, `mm-cn`, `writing` category) to use M2.7 as the new default.

## Bug Fixes

- **Dashboard Project Discovery** — Complete rewrite of project `cwd` resolution:
  - Removed unreliable folder name decoding (breaks on project names with dashes like `oh-my-claude` → `/oh/my/claude`)
  - JSONL search now checks root-level files, UUID subdirectories, and `subagents/` folders (newer Claude Code format)
  - Discovered projects jumped from 2 → 6 on test machine
  - Applied to both memory and preferences scanners

- **Global Memory Delete 404** — `resolveMemoryPath` for global scope only checked `notes/` subdirectory. Session memories stored in `sessions/` returned 404 on delete. Now checks both `notes/` and `sessions/` with fallback to `notes/` for new files.

- **Dashboard AI Operations Timeout** — `mem-daily` and other AI operations on the Settings page failed with "signal is aborted without reason" for large session memories. Increased `AbortSignal.timeout` from 60s to 180s to handle AI completions with substantial content.

- **Menubar Search Path Order** — Fixed `omc menubar` failing with "missing tauri.conf.json" error. Search paths now prioritize installed version (`~/.claude/oh-my-claude/apps/menubar`) over source directories, ensuring the complete installed copy is always used.

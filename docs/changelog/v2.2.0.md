# v2.2.0 - 2026-03-12

> Major release: Native Coworker Runtime, Unified API, Deep Modularization

---

## Highlights

### Native Coworker Runtime (Codex + OpenCode)

oh-my-claude now includes a fully native coworker runtime supporting both **OpenAI Codex** and **OpenCode** as first-class execution targets. This represents a fundamental shift from external process management to integrated, cross-platform coworker orchestration.

**Key Capabilities:**
- **Unified `coworker_task()` API** — Single interface for all coworker operations with 9 actions: `send`, `review`, `diff`, `fork`, `approve`, `revert`, `cancel`, `status`, `recent_activity`
- **Cross-platform viewer** — Unified tmux/WezTerm/Terminal.app viewer with automatic terminal detection and graceful fallbacks
- **Task lifecycle management** — Full support for task cancellation, interrupt handling, TUI toast notifications, viewer auto-close, and live status tracking
- **OpenCode plugin agent support** — Fuzzy matching against live `/agent` list, including plugin agents exposed by the server
- **Codex approval policies** — Native support for `never`, `on-request`, `on-failure`, `untrusted`, `strict`, `manual`, `full-auto` policies via `OMC_CODEX_APPROVAL_POLICY`

### Scoped Code Review

New `paths` parameter enables focused git diff reviews for specific file paths or directories. Combined with dynamic timeout estimation based on diff size and file count, this provides efficient, targeted code review workflows.

### Deep Modularization (4 Waves)

The coworker subsystem underwent 4 major refactoring waves:

1. **Wave 1** — Core protocol types, agent routing, basic spawn infrastructure
2. **Wave 2** — Unified viewer, task cancellation, status tracking
3. **Wave 3** — Codex app-server integration, OpenCode agent resolution, approval policies
4. **Wave 4** — Shell-aware viewer command resolution, Windows Git Bash wrapping, log rendering fixes

### Windows WezTerm Support

Complete Windows WezTerm integration for both debug proxy panes and coworker viewers:
- **Debug mode**: Coordinator script pattern for guaranteed mux access with [CC 65% | Proxy 35%] pane layout
- **Coworker viewer**: Native Git Bash wrapping for proper terminal handling and exit codes
- **Shell-aware resolution**: Automatic detection of zsh > Git Bash > PowerShell preference chain

## Features

### Native Coworker Runtime

- **Codex CLI integration** — Direct Codex CLI spawning with OAuth credential forwarding
- **OpenCode CLI integration** — OpenCode agent execution with live agent list resolution
- **Unified viewer** — Single viewer implementation supporting tmux (Unix), WezTerm (cross-platform), and Terminal.app (macOS)
- **Viewer auto-close** — Automatic viewer cleanup on task completion/cancellation
- **TUI toasts** — Toast notifications for task state changes
- **Live status tracking** — Real-time coworker status in statusline segment

### Unified Coworker API

- **`coworker_task(action="send")`** — Dispatch tasks to Codex/OpenCode with full context
- **`coworker_task(action="review")`** — Scoped code review with `paths` parameter
- **`coworker_task(action="diff")`** — Focused diff generation for specific paths
- **`coworker_task(action="fork")`** — Task forking for parallel execution
- **`coworker_task(action="approve")`** — Approval workflow with rich metadata
- **`coworker_task(action="revert")`** — Revert operations with change tracking
- **`coworker_task(action="cancel")`** — Graceful task cancellation
- **`coworker_task(action="status")`** — Structured worker status queries
- **`coworker_task(action="recent_activity")`** — Recent activity feed

### OpenCode Enhancements

- **Plugin agent support** — Fuzzy matching against live `/agent` endpoint
- **Explicit overrides** — Support for `agent`, `provider_id`, `model_id` parameters
- **Agent resolution** — Native agents and server-exposed plugin agents both accessible

### Codex Enhancements

- **Approval policies** — Full support for all Codex approval policies
- **Environment override** — `OMC_CODEX_APPROVAL_POLICY` environment variable
- **Rich metadata** — `decisionOptions`, `questions`, `details` in approval flows

### Proxy & Session Management

- **Proxy passthrough simplification** — Streamlined routing logic (~150 lines removed)
- **5-priority routing chain** — Directive(1) → Model-driven(2) → Session(3) → Global(4) → Passthrough(5)
- **Session isolation** — Each `omc cc` session has independent proxy instance
- **Debug mode** — `-debug` flag with visible proxy pane and logging

### Terminal Integration

- **WezTerm config** — `oh-my-claude wezterm-config` with 50k scrollback, JetBrains Mono, Dracula theme
- **tmux config** — `oh-my-claude tmux-config` with mouse mode, vi copy mode
- **Shell auto-detection** — Windows: zsh > Git Bash > PowerShell priority chain

### Memory System

- **AI-assisted compaction** — `/omc-mem-compact` for intelligent memory merging
- **AI-powered cleanup** — `/omc-mem-clear` for selective memory removal
- **Timeline consolidation** — `/omc-mem-summary` for date-range summarization
- **Three-tier search** — Hybrid (FTS5+vector) → FTS5 → Legacy fallback

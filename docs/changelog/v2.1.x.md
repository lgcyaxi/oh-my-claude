# v2.1.x Changelog

## v2.1.3 (2026-03-03 — 2026-03-12)

### Features

- **Native coworker runtime** — Full OpenCode + Codex integration with unified viewer, task lifecycle (send, review, diff, fork, approve, revert), cross-platform terminal spawning (tmux, WezTerm, macOS Terminal, xterm), and structured approval system with rich metadata
- **Windows WezTerm debug mode** — Coordinator script pattern for reliable pane splitting, proxy-first spawn with `wezterm cli`, inside-WezTerm vs outside-WezTerm routing, and Git Bash discovery for Windows (layout-agnostic directory walking)
- **Proxy response capture** — SSE capture stream with per-session ring buffer, `/response` long-poll endpoint, `/stream` real-time SSE endpoint, and proxy-mediated response capture for bridge workers
- **Bridge worker management** — `bridge_up`/`bridge_down` MCP tools for mid-session worker spawning, pre-configured provider switch at spawn time, async mutex for state file safety, and persistent state at `~/.claude/oh-my-claude/`
- **CLI tools and MCP installer** — `omc tools install` for uv, Bun, Rust, Go, Ollama, opencode, codex, wezterm, tmux; `omc tools mcp` for MiniMax (international + CN), ZhiPu, sequential-thinking; env key status inline
- **Ollama embedding management** — `omc m ollama` subcommand for pulling embedding models, checking dimensions, and auto-configuring memory semantic search
- **Codex app-server daemon** — Headless JSON-RPC 2.0 driver for `codex app-server` (no tmux/wezterm dependency), with generated TypeScript types, streaming text via notifications, and auth check on start

### Fixes

- **Windows Git Bash discovery** — `resolveNativeBash()` walks directory tree from `git.exe` to find `bin/bash.exe`, fixing WezTerm coworker viewer blank sessions and exit code issues
- **Inside-WezTerm debug pane** — Fixed proxy debug pane (right 35%) missing when `omc cc -debug` runs inside an existing WezTerm session
- **Bun TransformStream chain bug** — Manual reader-based piping replaces `pipeThrough()` to work around Bun 1.x silently dropping `transform()` invocations in chained TransformStreams
- **MCP scope detection** — Replaced `claude mcp list` (local-scope only) with direct `~/.claude.json` read to correctly detect user-scoped MCP servers
- **Menubar ENOEXEC on tarball installs** — Detects Git LFS pointer files and auto-downloads real binaries from GitHub raw URL
- **Bridge state race condition** — Async mutex locking prevents concurrent `bridge_up` calls from corrupting state file
- **Node.js v12 compatibility** — Fixed nullish coalescing in `bin/oh-my-claude.js` for WSL2's older Node.js
- **Codex viewer log rendering** — Tool activity entries now collapse consecutive same-prefix lines instead of showing raw output

### Refactoring

- **CLI modularization** — Split 3230-line monolith into 12 command modules across 4 domains: `core/` (install, doctor, update), `session/` (cc, proxy, auth), `tools/` (install, mcp), `manage/` (pref, memory, config, statusline, terminal-config, style, ollama, cleanup)
- **`src/` structural reorganization** — 20+ flat folders consolidated into 8 top-level domains (workers, assets, integration, shared, cli, mcp, coworker, proxy) with `git mv` for history preservation
- **Bun path resolution** — Extracted `resolveBunPath()` from 4 duplicated implementations into shared `src/cli/utils/bun.ts` with Windows-specific path handling and Chocolatey shim filtering
- **Unified coworker viewer** — Shared `spawnCoworkerViewer()` extracted from Codex-specific implementation; Codex viewer reduced from ~145 to ~13 lines
- **Manager command hierarchy** — Grouped infrequent commands under `omc manage|m` (`pref`, `memory`, `config`, `statusline`, `terminal-config`, `style`, `ollama`, `cleanup`); deprecated aliases print migration warnings

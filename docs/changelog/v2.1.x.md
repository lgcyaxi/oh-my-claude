## v2.1.3-beta.75 - 2026-03-12

### Windows WezTerm Coworker Viewer Fix

Fixed two bugs on Windows with WezTerm coworker viewer panes: (1) Codex viewer exits with code 1 ("Process didn't exit cleanly" warning), (2) OpenCode TUI shows blank session. Root cause: `spawnWezTerm()` used `cmd.exe` on Windows while macOS used `bash -lc`.

**`src/coworker/viewer.ts`** — Added `resolveNativeBash()` for Windows Git Bash discovery:
- Discovers native Git Bash (`bash.exe`) via candidate paths + `where git` directory walking
- Walks up the directory tree from git binary to find `bin/bash.exe` — layout-agnostic, works with non-standard Git installations (e.g., `D:\Applications\Git\mingw64\bin\git.exe`)
- Result is cached for the process lifetime with `_resetNativeBashCache()` for testing
- Modified `spawnWezTerm()` Windows branch to use Git Bash with `-lc` wrapping, matching macOS/tmux behavior

**`src/coworker/codex/viewer.ts`**:
- Removed `& exit /b 0` Windows workaround for exit code masking
- Unified to single platform-agnostic command

**`src/coworker/opencode/viewer.ts`**:
- Removed Windows platform branching (`cmd.exe /d /c` path)
- Unified `|| opencode` fallback across all platforms

### Codex Viewer Log Rendering Fix

**`src/cli/commands/manage/coworker-log.ts`**:
- Fixed `tool_activity` entries showing raw code lines (`TOOL: command output\n<line>`) instead of collapsing
- Added `toolActivityPrefix()` to extract descriptive prefix before `\n` for merge comparison
- Consecutive tool_activity entries with same prefix (e.g. "command output") now collapse to `TOOL: command output ×N`
- `formatEntry()` strips raw output tail, showing only the descriptive prefix

**`tests/coworker/viewer.test.ts`**:

## v2.1.3-beta.55 - 2026-03-10

### Added

- **Codex Task Cancellation & Turn Interrupt** — `cancelTask()` now calls `daemon.interruptActiveTurn()` to send a real `turn/interrupt` JSON-RPC call to Codex, not just return false. - **OpenCode Agent Selection** — `native` field added to `OpenCodeAgentEntry` interface. Prefers native agents (build/general/explore) over summary-style primary agents. `OMC_OPENCODE_AGENT` env var override.

- **Statusline Segments DRY (continued)** — `segments/index.ts`: `applyPreset()` now uses imported `ALL_SEGMENT_IDS` instead of hand-written array (~15 lines removed).

  - Exported from `src/workers/daemon/ais/index.ts`

## v2.1.3-beta.1 - 2026-03-03

### Added

- **Proxy Response Capture (Phase 1)**: New `src/proxy/response-cache.ts` module captures API responses during SSE streaming. Per-session ring buffer (max 5 entries) stores completed responses with text, provider, model, usage tokens, and sequence numbers. `SSECaptureTransformStream` parses Anthropic-format SSE events as passthrough (side-effect only, no latency added). `wrapWithCapture()` intercepts switched/directive responses for capture.

- **Control API `/response` Endpoint**: `GET /response?session=ID&seq=N&wait=true&timeout=MS` returns captured responses with long-poll support (500ms intervals, max 120s). Sequence filtering avoids stale reads. Returns `{found: true, response: {seq, text, provider, model, completedAt, streaming, usage}}`.

- **Control API `/stream` Endpoint (Phase 3)**: `GET /stream?session=ID&timeout=MS` SSE endpoint for real-time response streaming. Emits `connected`, `delta` (text deltas), `done` (completion with metadata), and `timeout` events. Uses event listener registry (`addCaptureListener`/`removeCaptureListener`) for real-time notifications.

- **Bridge State Path Migration**: Bridge state file moved from `tmpdir()/oh-my-claude-bridge/` to `~/.claude/oh-my-claude/bridge-state.json`. Persistent across reboots, easier to inspect and manage.

- **Bridge State Extended with Proxy Info**: `AIEntry` interface now includes `proxySessionId` and `proxyControlPort` fields. Auto-populated after `--switch` succeeds in `bridge up` command.

- **Proxy-Mediated Response Capture for Bridge Workers**: `bridge_send` MCP tool now tries proxy capture first (via `pollProxyResponse()`) before falling back to pane-scrollback polling. Clean API response text without TUI chrome pollution. Works for Anthropic-native providers (GLM, DeepSeek, Aliyun).

- **Bridge Mode Token Optimization**: Hook-based bridge mode constraint injection now uses short reminder (~15 tokens) after first full injection (~150 tokens) per session. Uses `bridge-injected.flag` file in session directory to track. Saves ~135 tokens per turn after the first prompt.

- **`bridge_up` / `bridge_down` MCP Tools (Phase 2)**: New MCP tools for mid-session bridge worker management. `bridge_up(name, switch?)` spawns CC worker with auto-switch, `bridge_down(name)` kills worker and cleans up state. Enables dynamic worker spawning without restarting Claude Code session.

- **`stream` Parameter on `bridge_send`**: Forward-compatibility addition for future streaming support. Schema-only for now (MCP tool calls are blocking).

### Fixed

- **Bun TransformStream Chain Bug**: `wrapWithCapture()` now uses manual reader-based piping instead of `pipeThrough()` to avoid Bun 1.x bug where chained TransformStreams silently drop `transform()` invocations. Critical for double-transform paths (OpenAI→Anthropic converter → capture).

- **OpenAI-Format Provider Capture**: Integrated capture directly into `createOpenAIToAnthropicResponse()` for Kimi/MiniMax providers. Parses `content_block_delta` events during conversion to extract text. Requires proxy restart to activate.

### Changed

- **Hook Memory Injection**: `memory-awareness.ts` bridge mode prefix now conditionally injects full constraints on first prompt, then short reminder on subsequent prompts. Dynamic mode.json detection alongside `OMC_BRIDGE_MODE` env var for activation.

- **Handler Response Flow**: Refactored `handleDirectiveRoute` and `handleSwitched` to unify response construction before capture wrapping. All format paths (Responses API, OpenAI, Anthropic) now captured consistently.

## v2.1.3-beta.19 - 2026-03-04

### Added

- **`CodexAppServerDaemon`** â€” new headless daemon that drives `codex app-server` via JSON-RPC 2.0 over stdin/stdout. Replaces the terminal-pane + JSONL-polling architecture of `CodexDaemon` for the app-server path:
  - No tmux/wezterm dependency â€” fully headless, no terminal pane needed
  - No IPC FIFO channels, no `CodexStorageAdapter` file watching
  - Event-driven response via JSON-RPC notifications (not polled from disk)
  - Auth check on `start()` via `getAuthStatus` RPC â€” throws with clear message if unauthenticated; defers to codex's credential store (`~/.codex/auth.json`)
  - One conversation per daemon instance (`newConversation` + `addConversationListener` on start, reused for all turns)
  - Subscribes to conversation events via `addConversationListener` before sending turns â€” required to receive streaming notifications
  - Streams text via `codex/event/agent_message_delta` notifications, finalizes on `codex/event/task_complete`
  - `model` captured from `newConversation` response and forwarded to `sendUserTurn` (required â€” must be non-null)
  - Process crash handling: rejects all pending RPCs and surfaces error to next `checkResponse()` poll
  - Exported from `src/workers/daemon/ais/index.ts`

- **Generated codex app-server TypeScript types** (`src/workers/codex/`) â€” full protocol type definitions generated by `codex app-server generate-ts`, covering all JSON-RPC request/response/notification shapes for compile-time safety.

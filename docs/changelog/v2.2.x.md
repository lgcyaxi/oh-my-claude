# v2.2.x Changelog

## v2.2.13 (2026-04-13)

### Bug Fixes
- **Cross-project session leak** — `omc cc -r` (resume) used `readInstances()` from the global instance registry which returned sessions from ALL projects. Worse, it stored `controlPort` as `sessionId`, reusing a port number instead of a real session ID. Now uses the project-scoped proxy registry filtered by CWD
- **`/continue` shows no history** — Each `omc cc` launch generated a random session ID → unique `ANTHROPIC_BASE_URL` every time → Claude Code saw each launch as a fresh backend with no conversation history. Now derives deterministic session identity from CWD (SHA-256 hash → stable 8-char session ID + preferred port pair), so same project always gets the same URL

### Changes
- **Proxy reuse** — When a healthy proxy is already running for the same project directory, `omc cc` reuses it instead of spawning a new one (skip proxy spawn + port allocation)
- **`-n` / `--new` shortcut** — Force fresh session with random ID + ports (overrides deterministic identity)
- **psmux optional** — Terminal multiplexer (tmux/psmux) downgraded from warning to informational in `doctor` output; Claude Code's native teammate switching makes panels less critical on Windows
- **Preferred port allocation** — `findFreePorts()` now accepts preferred ports and tries them first before falling back to random; enables stable URLs when ports aren't occupied
- **tmux session reuse on Windows** — psmux now creates new windows in existing sessions instead of always creating new sessions; improved `killTerminalPane()` with kill-window → kill-session fallback
- **tmux attach fix** — Outside tmux, `omc cc` now finds which session owns the CC window and attaches correctly (was blindly attaching to a session name that might not exist)

## v2.2.12 (2026-04-13)

### Bug Fixes
- **`/resume` can't find sessions** — `sessions-index.json` entries created by the dashboard had `messageCount: 0` (hardcoded in `extractQuickMeta`), making Claude Code's `/resume` command treat them as empty. Added `countMessages()` to `parser.ts` that does a fast string-match scan of JSONL files; `handler.ts` and `ai-rename.ts` now call it when creating new index entries

## v2.2.10 (2026-04-13)
### Changes
- **`omc cc` tmux default** — Default terminal mode changed from `none` to `auto`; tmux/psmux auto-detection now works on Windows (removed platform guard), matching Unix behavior
- **`--continue`/`--resume` session reuse** — When Claude Code's `--continue` or `--resume` flag is used, omc reuses the last session's ID so `ANTHROPIC_BASE_URL` matches and Claude Code can find the prior conversation
- **Dashboard instance cleanup** — Dead proxy instances are now filtered from `/api/instances` response and deregistered from the registry; `readInstances()` also checks PID liveness for immediate cleanup instead of waiting for 5-min heartbeat TTL
- **`-nf` no-flicker shortcut** — New OMC shortcut `-nf` / `--no-flicker` sets `CLAUDE_CODE_NO_FLICKER=1` env var; threaded through all launch paths (inline, tmux inline, detached tmux, remote-control)
- **Dashboard session scanner** — Supports Claude Code's directory-based sessions (`{id}/subagents/`) and index-only archive sessions; previously only found flat `.jsonl` files, missing 90%+ of sessions
- **Session DELETE for archives** — Deleting sessions that only exist in `sessions-index.json` (no file on disk) now works; cleanup handlers also purge orphaned index entries
- **psmux bash config** — New `oh-my-claude terminal-config psmux` command configures Git Bash as psmux's default shell, fixing team agent spawn failures on Windows (Claude Code sends bash-style commands to panes)
- **Doctor psmux check** — Warns if psmux bash config is missing on Windows (team agents won't work without it)

### Bug Fixes
- **psmux bash syntax** — All tmux launch paths (detached, inline, send-keys) now use bash syntax instead of cmd.exe syntax (`export`/`unset` instead of `set`/`cd /d`); fixes broken psmux sessions when psmux is configured with Git Bash
- **tmux window not attached** — `omc cc` now auto-switches to the new tmux window (via `select-window` or `attach-session`) in all modes, not just debug mode
- **psmux window stays open after exit** — On Windows, `send-keys` spawns a bash shell that persists after Claude exits; now appends `; exit` so the psmux window closes automatically (matching Unix behavior)
- **BM25 search ranking** — `normalizeRank()` in `search.ts` and `hybrid-search.ts` always returned 1.0 for all results due to `Math.max(0, rank)` clamping negative BM25 scores; fixed to `Math.abs(rank)`
- **Memory hook read wrong file** — `getClaudeNativeMemory` read `CLAUDE.md` instead of `MEMORY.md`, also fixed Windows path normalization (backslash handling)
- **Token counter** — Stream converters (`openai-stream.ts`, `responses-stream.ts`) incremented token count per SSE chunk instead of estimating per token; now uses `Math.ceil(length/4)` with provider-reported usage override
- **`updateMemory` scope** — Missing `projectRoot` passthrough to `getMemory`, causing lookups from wrong directory
- **Module side effect** — `cleanupStaleSessions` ran at import time; wrapped in lazy init
- **Proxy state** — Replaced inline `require('node:fs')` with top-level import; used `AbortSignal.timeout()` instead of manual controller

### Dead Code Removed
- Deleted `src/shared/context/` module (5 files) and `context-profiles.ts` — zero consumers
- Removed legacy `session-logger.ts` hook (source, build script entry, hooks registry)
- Removed 4 unused provider client singletons, `callMemoryAIWithMessages`, `providerAgentMap`
- Removed dead `test:smoke:codex` script from package.json

### Code Quality
- Created `src/shared/utils.ts` with `toErrorMessage()` utility — replaced 25 duplicate error coercion patterns across 18 files
- Fixed unused variable warnings (`icon`, `shortHash` import, `index`, `delta`)
- Updated CLAUDE.md: 7 stale proxy paths, removed nonexistent `mcp/tasks/run.ts`, fixed machine-specific paths, removed `pend` command
- Fixed README cross-language links, removed ghost commands (`/omc-up`, `/omc-down`, `/omc-pend`), deduplicated `/omc-opencode`

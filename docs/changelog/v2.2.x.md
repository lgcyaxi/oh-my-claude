# v2.2.x Changelog

## v2.2.14-beta.2 (2026-04-25)

### Bug Fixes
- **Statusline context bar respects the switched provider model** — `context` segment now queries the proxy switch state and sizes the context window against the effective upstream model (fixes ~5x over-report on DeepSeek V4 / Qwen 3.6 1M sessions); DeepSeek V4 window corrected from 128K to 1M per official pricing docs
- **Context segment sources model from the transcript** — `parseTranscriptTail()` reads the assistant entry's `message.model` (the exact upstream billing identity, verified directly off a real `.jsonl`), so switched sessions no longer rely on a proxy control-API round-trip and native Claude picks up the dated `claude-opus-4-7-…` slug. Fixed Claude Opus 4.6 / 4.7 context windows to 1M (both ship with 1M at standard pricing), corrected `glm-5-turbo` to 128K, and added explicit `glm-4.6` (200K) / `glm-4.5` (128K) rows against the authoritative vendor docs

## v2.2.14-beta.1 (2026-04-25)

### Added
- **DeepSeek V4 Flash (haiku tier)** — `deepseek-v4-flash` is the lite, non-thinking V4 variant. Aliases `ds-f` / `deepseek-v4-flash`. Routed by the proxy when Claude Code issues a haiku-tier request while switched to DeepSeek
- **Claude-tier → provider-model mapping (DeepSeek + ZhiPu/Z.AI)** — proxy reads `claudeTierMap` from `models-registry.json` and rewrites Claude-tier requests transparently. Defaults: DeepSeek `opus → deepseek-v4-pro (effort=max)`, `sonnet → deepseek-v4-pro (effort=high)`, `haiku → deepseek-v4-flash (fast path)`; ZhiPu/Z.AI `opus → glm-5.1`, `sonnet → glm-5-turbo`, `haiku → glm-4.5-air`
- **GLM-4.5 Air** — added to ZhiPu and Z.AI rosters as the haiku-tier model. New alias `gair`
- **`resolveEffectiveRoute` + `resolveClaudeTier`** — new helpers in `src/proxy/routing/model-resolver.ts` returning `{ model, effort? }` per request, enabling tier-aware routing without env-var injection in the CC launcher

### Changed
- **DeepSeek sanitizer (V4 split paths)** — `sanitizeDeepSeekV4` now accepts `{ effort }` and picks between a thinking path (V4 Pro / opus / sonnet: force `thinking.type=enabled`, inject `output_config.effort` from the resolver, keep placeholder thinking blocks) and a fast path (V4 Flash / haiku: drop top-level `thinking`, strip thinking content blocks and unsupported types — mirrors the old `deepseek-chat` behaviour)
- **Switched / directive handlers** — `switched.ts` now calls `resolveEffectiveRoute` and forwards `effort` to the sanitizer; `directive.ts` derives effort from the target model (`deepseek-v4-pro → max`, `deepseek-v4-flash → fast path`)
- **Statusline / menubar fallback / `/omc-switch` / `omc cc` picker / README (EN + zh-CN)** — refreshed to advertise DeepSeek V4 Pro + V4 Flash, the GLM tier lineup, and the proxy-side tier mapping

### Removed
- **Legacy DeepSeek names (`deepseek-chat` / `deepseek-reasoner`)** — hard-removed from aliases, statusline display maps, `omc cc` proxy picker, and `/omc-switch` docs. Upstream sunsets these IDs on 2026-07-24; the proxy no longer forwards or recognises them

### Bug Fixes
- **`omc cc` on Windows PowerShell** — inline and remote-control launch paths spawned `claude` through `process.env.SHELL || 'bash'`, which on PowerShell often resolves to `/bin/bash` and cannot find `claude.cmd`/`claude.exe` (bash PATH lookup ignores `PATHEXT`). Fixed via a new `resolveCCShell()` helper in `src/cli/commands/session/cc-routing.ts` that returns `shell: true` on native Windows (so Node spawns via `cmd.exe`) and honours `$SHELL || 'bash'` everywhere else, including Git Bash (`MSYSTEM`). Both spawn sites in `cc-session-unix.ts` and `runNativeCC` now use it
- **Statusline context segment was always measuring against Claude's window** — `collectContextData` read `claudeCodeInput.model.id`, which stays `claude-opus-4-6` after a proxy switch. DeepSeek V4 (1M) and Qwen 3.6 (1M) sessions were divided by 200K and over-reported usage ~5x; GLM-5.1 / GLM-5 only looked correct because they coincidentally match Claude's 200K. The segment now resolves the effective upstream model via the proxy control API (session-scoped) with a file-state fallback — same pattern as the model segment — and falls back to Claude's ID only when no proxy session is active
- **DeepSeek context window entry was stale (128K)** — `CONTEXT_WINDOWS` in `src/statusline/segments/context.ts` hard-coded `deepseek: 128_000`. Per DeepSeek's official pricing docs (https://api-docs.deepseek.com/zh-cn/quick_start/pricing) both `deepseek-v4-pro` and `deepseek-v4-flash` have a **1M** context length. Replaced with explicit `deepseek-v4-pro` / `deepseek-v4-flash` entries plus a generic `deepseek: 1_000_000` fallback; also added explicit `glm-5-turbo` (200K) and `glm-4.5-air` (128K) rows to keep substring-based lookup unambiguous
- **Context segment now reads the model straight off the transcript** — `parseTranscriptTail()` returns `{ usage, model }` from the last assistant entry. Claude Code mirrors the raw upstream response (e.g. `"message":{"model":"qwen3.6-plus","usage":…}` — line 15 of a real transcript), so `message.model` is exactly what the provider billed the response against. Resolution priority is now **transcript → stdin → switch-state warm start → default**; the proxy control API round-trip is only used when the transcript is empty (first turn), making the common path free of network I/O and correct even for native Claude sessions where stdin doesn't carry the dated model slug
- **Claude Opus 4.6 / 4.7 context window was capped at 200K** — Opus 4.6 and the new Claude Code default (Opus 4.7) both ship with a **1M** token context window at standard pricing per [Anthropic's release notes](https://platform.claude.com/docs/en/about-claude/models/whats-new-claude-4-7). Updated `CONTEXT_WINDOWS` to `claude-opus-4-{6,7}: 1_000_000`; Sonnet / Haiku 4.6 / 4.7 kept at 200K (no documented 1M for those tiers)
- **GLM context windows corrected against authoritative vendor docs** — `glm-5-turbo` was wrongly pinned at 200K; per ZhiPu's 对比表 ([aipuzi.cn/ai-news/glm-5-1.html](https://www.aipuzi.cn/ai-news/glm-5-1.html)) it is **128K**. Added explicit `glm-4.6: 200_000` ([docs.z.ai/guides/llm/glm-4.6](https://docs.z.ai/guides/llm/glm-4.6)) and `glm-4.5: 128_000` ([docs.z.ai/guides/llm/glm-4.5](https://docs.z.ai/guides/llm/glm-4.5)) rows above the `glm-4` generic fallback so substring matching lands on the right window

## v2.2.13 (2026-04-13)

### Bug Fixes
- **Cross-project session leak** — `omc cc -r` (resume) used `readInstances()` from the global instance registry which returned sessions from ALL projects. Worse, it stored `controlPort` as `sessionId`, reusing a port number instead of a real session ID. Now uses the project-scoped proxy registry filtered by CWD
- **`/continue` shows no history** — Each `omc cc` launch generated a random session ID → unique `ANTHROPIC_BASE_URL` every time → Claude Code saw each launch as a fresh backend with no conversation history. Now derives deterministic session identity from CWD (SHA-256 hash → stable 8-char session ID + preferred port pair), so same project always gets the same URL

### Changes
- **Proxy reuse** — When a healthy proxy is already running for the same project directory, `omc cc` reuses it instead of spawning a new one (skip proxy spawn + port allocation)
- **`-n` / `--new` shortcut** — Force fresh session with random ID + ports (overrides deterministic identity)
- **psmux optional** — Terminal multiplexer (tmux/psmux) downgraded from warning to informational in `doctor` output; Claude Code's native teammate switching makes panels less critical on Windows
- **Preferred port allocation** — `findFreePorts()` now accepts preferred ports and tries them first before falling back to random; enables stable URLs when ports aren't occupied
- **tmux session reuse on Windows** — psmux now creates new windows in existing sessions instead of always creating new sessions; improved `killTerminalPane()` with kill-window → kill-session fallback
- **tmux attach fix** — Outside tmux, `omc cc` now finds which session owns the CC window and attaches correctly (was blindly attaching to a session name that might not exist)

## v2.2.10 (2026-04-13)
### Changes
- **`omc cc` tmux default** — Default terminal mode changed from `none` to `auto`; tmux/psmux auto-detection now works on Windows (removed platform guard), matching Unix behavior
- **`--continue`/`--resume` session reuse** — When Claude Code's `--continue` or `--resume` flag is used, omc reuses the last session's ID so `ANTHROPIC_BASE_URL` matches and Claude Code can find the prior conversation
- **Dashboard instance cleanup** — Dead proxy instances are now filtered from `/api/instances` response and deregistered from the registry; `readInstances()` also checks PID liveness for immediate cleanup instead of waiting for 5-min heartbeat TTL
- **`-nf` no-flicker shortcut** — New OMC shortcut `-nf` / `--no-flicker` sets `CLAUDE_CODE_NO_FLICKER=1` env var; threaded through all launch paths (inline, tmux inline, detached tmux, remote-control)
- **Dashboard session scanner** — Supports Claude Code's directory-based sessions (`{id}/subagents/`) and index-only archive sessions; previously only found flat `.jsonl` files, missing 90%+ of sessions
- **Session DELETE for archives** — Deleting sessions that only exist in `sessions-index.json` (no file on disk) now works; cleanup handlers also purge orphaned index entries
- **psmux bash config** — New `oh-my-claude terminal-config psmux` command configures Git Bash as psmux's default shell, fixing team agent spawn failures on Windows (Claude Code sends bash-style commands to panes)
- **Doctor psmux check** — Warns if psmux bash config is missing on Windows (team agents won't work without it)

### Bug Fixes
- **psmux bash syntax** — All tmux launch paths (detached, inline, send-keys) now use bash syntax instead of cmd.exe syntax (`export`/`unset` instead of `set`/`cd /d`); fixes broken psmux sessions when psmux is configured with Git Bash
- **tmux window not attached** — `omc cc` now auto-switches to the new tmux window (via `select-window` or `attach-session`) in all modes, not just debug mode
- **psmux window stays open after exit** — On Windows, `send-keys` spawns a bash shell that persists after Claude exits; now appends `; exit` so the psmux window closes automatically (matching Unix behavior)
- **BM25 search ranking** — `normalizeRank()` in `search.ts` and `hybrid-search.ts` always returned 1.0 for all results due to `Math.max(0, rank)` clamping negative BM25 scores; fixed to `Math.abs(rank)`
- **Memory hook read wrong file** — `getClaudeNativeMemory` read `CLAUDE.md` instead of `MEMORY.md`, also fixed Windows path normalization (backslash handling)
- **Token counter** — Stream converters (`openai-stream.ts`, `responses-stream.ts`) incremented token count per SSE chunk instead of estimating per token; now uses `Math.ceil(length/4)` with provider-reported usage override
- **`updateMemory` scope** — Missing `projectRoot` passthrough to `getMemory`, causing lookups from wrong directory
- **Module side effect** — `cleanupStaleSessions` ran at import time; wrapped in lazy init
- **Proxy state** — Replaced inline `require('node:fs')` with top-level import; used `AbortSignal.timeout()` instead of manual controller

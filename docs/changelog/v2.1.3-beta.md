# v2.1.3-beta Changelog

> Beta releases from 2026-03-03 to 2026-03-05

## v2.1.3-beta.43 - 2026-03-05

### Model-driven auto-routing + agent architecture overhaul

- **feat(proxy)**: Model-driven auto-routing (Priority 2) — when a request contains a non-Claude model ID (e.g. `model: "qwen3.5-plus"`), proxy automatically routes to the correct provider. New `resolveModelToProvider()` in `model-resolver.ts` with reverse `modelToProviders` map built from models-registry.json. Routing priority: directive(1) → auto-route(2) → session(3) → global(4) → passthrough(5). Skips Claude models, picks first configured provider for multi-provider models
- **feat(agents)**: Auto-routing agent presets — agent YAML frontmatter now emits `model` field for non-Claude agents (triggers proxy auto-routing). Bridge agents converted to native: analyst (bridge/deepseek-chat → native/`qwen3.5-plus`), librarian (bridge/glm-5 → native/`glm-5`), document-writer (bridge/MiniMax-M2.5 → native/`MiniMax-M2.5`). New agents registered: oracle (`claude-sonnet-4-6`, passthrough), navigator (`kimi-for-coding`), hephaestus (`kimi-for-coding`). `bridgeAgents` deprecated (empty). All 11 agents in `taskAgents`/`nativeAgents`
- **refactor(mcp)**: Move from schemas to register pattern — tool definitions use inline `register()` calls instead of centralized schema objects
- **feat(memory)**: Structured category system and menubar picker
- **refactor(proxy)**: Restructure into domain-aligned subdirectories — `src/proxy/` reorganized into `handlers/`, `routing/`, `state/`, `streaming/`
- **refactor**: Flatten `src/integration/` into domain-aligned top-level dirs
- **feat(proxy)**: Canonical provider alias system with CLI integration
- **feat(preferences)**: Short hash IDs with prefix resolution and tag matching
- **refactor(statusline)**: Replace `metadata.newLine` with config-driven row layout
- **refactor(doctor)**: Split `doctor.ts` into modular directory structure

- **fix(proxy)**: Alias-to-provider resolution — CLI aliases like `zp` now correctly resolve to config provider names like `zhipu` at proxy startup (`server.ts`)
- **fix(proxy)**: Per-process default switch state — `setDefaultSwitchState()` replaces global file write for bridge worker pre-switch, eliminating session contamination and session ID mismatch
- **fix(proxy)**: `/status` endpoint checks process default when no session match, ensuring worker statusline shows correct switched model
- **fix(proxy)**: Startup log now reads process default state instead of global file
- **fix(bridge)**: Bus path disabled for all workers — Pane-based (`cc:*`) and proc-based (`codex`) both use legacy path for reliable auto-spawn + delivery
- **fix(bridge)**: PostToolUse bus polling — Bridge workers poll bus on every tool use for task delivery (defense-in-depth)

## v2.1.3-beta.33 - 2026-03-04

### fix(cc): restore per-session proxy when no global proxy is running

- **Root cause**: After routing refactor, non-bridge mode without global proxy fell through to `runNativeCC` (no proxy)
- **Fix**: Added 4-condition routing:
  1. Bridge mode → per-session proxy
  2. Global proxy + terminal → reuse global proxy
  3. No global proxy + terminal → spawn per-session proxy **(new)**
  4. No terminal → native launch

## v2.1.3-beta.32 - 2026-03-04

### fix(statusline): quota display for Kimi, MiniMax, and Aliyun

- **Root cause**: Dynamic `require()` calls in Bun-bundled statusline replaced with throws
- **Fixes**:
  - Replaced dynamic requires with static imports for credential detection
  - Kimi: compute `used = limit - remaining` when API returns only `remaining`
  - MiniMax: Strategy 1 — fetch quota with API key only (no GroupId required)
  - Provider registry: fix `isConfigured` for MiniMax/Kimi, add Aliyun entry

### fix(cc): proxy not applied on `oh-my-claude cc` with global proxy

- **Root cause 1**: `isProxyConfigured()` required `/s/` in URL; global proxy has no `/s/`
- **Root cause 2**: `spawnProxySwitchedCC()` set env var but never called `/switch` endpoint
- **Fixes**:
  - Accept any localhost ANTHROPIC_BASE_URL
  - Prepend `curl -X POST /switch` to spawned shell command
